# RTL Design AI Skill: 需求理解与架构方案

## 目的
基于设计需求规范，与AI助手协作进行**架构设计和方案评估**，输出结构化的设计方案。

---

## 前置条件
- [ ] 设计需求规范已使用模板填写完整
- [ ] 所有相关IP文档已收集
- [ ] 编码规范/Review Checklist 已准备

---

## 📋 核心工作流

### 步骤 1: 需求信息输入

**工程师输入给AI**（直接复制设计需求模板）：
```
你是一名资深RTL数字IC设计工程师。现在我需要你辅助完成一个项目的设计方案。

以下是设计需求规范：

[完整粘贴设计需求模板的内容]

请执行以下任务：
1. 分析需求的完整性和一致性，指出任何不清楚的地方
2. 基于需求和系统约束，给出详细的架构设计方案
3. 评估选中的IP是否满足需求
4. 提出可能的风险点和优化方向
```

### 步骤 2: AI架构方案生成

**AI输出内容**（期望包括）：
```
✓ 需求理解总结
✓ 推荐架构设计 + 设计思路说明
✓ 模块划分与接口定义
✓ 时序分析（关键路径、频率可达性）
✓ IP评估报告
✓ 风险分析和缓解方案
✓ 实现策略建议
```

### 步骤 3: 工程师架构评审

**检查清单**：
- [ ] 架构是否满足功能需求？
- [ ] 接口定义是否清晰且完整？
- [ ] IP选择是否合理？有其他替代方案吗？
- [ ] 时序可行性？是否需要流水线/并行化处理？
- [ ] 面积和功耗预估合理吗？
- [ ] 有没有遗漏的特殊场景或约束？

**反馈给AI**：
```
我对方案有以下意见：

1. [修改点1]：原因是xxxx，建议改成xxxx
2. [修改点2]：...
3. [需要澄清的地方]：xxxxx

同时，我提供补充信息：
- xxxx
- xxxx

请基于上述反馈修改架构方案。
```

### 步骤 4: AI方案迭代

AI根据反馈重新生成完整的架构方案。重复步骤3-4直到方案得到认可。

### 步骤 5: 方案冻结与交付

确认方案后，请AI生成**最终架构设计文档**：
```
请生成最终的架构设计文档，包括以下章节：

1. 功能架构概述
2. 模块划分和接口定义表
3. 数据通路设计
4. 控制路径设计
5. 时序分析汇总
6. IP集成方案
7. 风险和缓解措施
8. RTL设计建议（包括推荐的编码模式）

输出格式：Markdown，方便后续参考。
```

## ⚠️ 常见问题和处理

### 问题1: 架构方案不满足时序需求
**症状**：AI给出的架构在目标频率下无法实现
**处理**：
1. 告诉AI具体的频率约束和关键路径分析
2. 要求AI添加流水线阶段或并行处理
3. 讨论不同的权衡方案（面积 vs 性能）

### 问题2: IP库中没有完全匹配的IP
**症状**：需要的功能无现成IP
**处理**：
1. 让AI评估现有IP的变通方案
2. 或建议自研该IP
3. 讨论时间/面积成本

### 问题3: 需求本身不清晰或有矛盾
**症状**：AI指出需求中的矛盾之处
**处理**：
1. 与架构师沟通澄清需求
2. 更新设计需求模板
3. 重新触发AI分析

## 💡 最佳实践

1. **提供具体的数字**：不要说"高性能"，要说"需要300 MHz，延迟<10 cycles"
2. **早期澄清异常情况**：边界条件、错误处理要在架构阶段明确
3. **IP评估要严格**：不满足的IP可能导致后续大量返工
4. **保留迭代空间**：架构阶段的改动成本最低
5. **记录决策理由**：为后续RTL设计提供指导

---

**技巧**：架构评审阶段是最关键的节点。投入足够的时间在这里，可以大幅减少后续RTL和验证的工作量。
