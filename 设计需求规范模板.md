## 📝 编码规范

参考 [RTL编码指南](../guidelines/RTL_Coding_Style.md)

关键要求摘要：
```
- 命名规范：
  - 模块名/文件名：snake_case + _v版本号
  - 信号名：snake_case
  - 参数名：大写 PARAMETER_NAME
  
- 时序设计：
  - 同步设计优先
  - 所有 flip-flop 的 D 端输入必须来自组合逻辑
  
- 复位设计：
  - 统一使用 active-low 异步复位
  - 所有 flip-flop 都必须有复位信号
  
- 其他关键点：
  - 避免 latch
  - 避免跨时钟域直接连接
  - 模块间接口使用参数化
```

- [ ] 工程师审查方案，提出修改意见

### Phase 3: 自仿真与验证

- [ ] 性能评估和优化建议

- [ ] 生成设计文档

## 附录：AI助手使用建议

### 何时需要与AI互动
```
✓ 需要架构方案时
✓ 需要大量重复代码时
✓ 需要自动生成testbench时
✓ 需要性能评估时

✗ 不建议：仅让AI做简单翻译或小改动
```

### 如何提高AI输出质量
```
1. 提供尽可能详细的背景信息
2. 在需求中包含具体例子
3. 明确指出限制条件
4. 逐步迭代而非一次性要求完整设计
5. 及时反馈和修正方向
```
